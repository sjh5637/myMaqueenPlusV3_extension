# 큐브 기준 거리/각도 보정-테스트 스크립트 설계

## 배경 / 목적

`AUTONOMOUS_WANDER_EXAMPLE.md`의 메인 자율주행 코드는 빈 공간을 기준으로
하는 `바닥보정()`만 가지고 있다. 실제 거리 정답(ground truth)이 없어서
"라이다가 실제로 몇 mm를 몇 mm로 읽는지", "센서가 인식 가능한 최소 거리는
몇 mm인지", "값이 비정상적으로 튀는 지점이 있는지"를 정량적으로 알 수
없다.

5x5x5cm 나무 정육면체를 바닥에 고정해두고, 로봇이 그 큐브에 코를 맞댄
상태(거리 0)에서 시작해 1cm씩 정확히 후진하며 라이다(매트릭스 8x8)와
전면 초음파(P13 TRIG / P14 ECHO)를 동시에 측정하면, 알려진 실제 거리와
센서 값을 직접 비교할 수 있다. 이 설계는 메인 주행 코드와는 완전히
분리된, 1회성 벤치 테스트용 스크립트를 다룬다.

## 비목표 (Out of scope)

- 메인 자율주행 코드(`AUTONOMOUS_WANDER_EXAMPLE.md`)는 수정하지 않는다.
- 라이다 높이(A+B) 자동 스윕은 하지 않는다 — 필요하면 높이만 바꿔 같은
  테스트를 재실행한다.
- LCD 레이더맵 등 무거운 화면 갱신은 넣지 않는다 — 측정 정밀도/속도가
  우선이며, 5x5 LED 매트릭스로 진행 상황 숫자만 표시한다.

## 하드웨어 전제

- 로봇 정면에 5x5x5cm 나무 정육면체를 바닥에 고정(로봇에는 붙이지 않음).
- 로봇을 큐브에 정면으로, 가능한 한 똑바로 맞대어 거리 0 위치에 둔다.
- 매트릭스 라이다: `matrixLidarDistance.Addr.Addr4` (0x33), 8x8 그리드.
- 전면 초음파: TRIG = `DigitalPin.P13`, ECHO = `DigitalPin.P14`
  (`maqueenPlusV2.readUltrasonic(P13, P14)`, 단위 cm).
- 라디오: 그룹 77 (메인 코드의 디버그 콘솔과 동일 채널/리시버 재사용 가능).

## 동작 흐름

1. 부팅 시 `로봇초기화_테스트()`: I2C init, 라이다 init, 라디오 그룹 77,
   송신 출력 설정. 5x5 매트릭스에 대기 아이콘 표시.
2. B 버튼을 누르면 시작.
3. 3, 2, 1 카운트다운 (5x5 매트릭스 숫자 표시).
4. 스텝 0 ~ 30 (cm, 총 31스텝)을 순회:
   - 스텝 0: 후진 없이 그 자리에서 측정 (큐브가 거의 붙어있는 상태 —
     센서 최소 인식 거리 추정용. 0/무효값이 나올 수 있음을 정상으로
     간주).
   - 스텝 1~30: `maqueenPlusV2.pidControlDistance(SpeedCCW, 1,
     MyInterruption.NotAllowed)`로 정확히 1cm 후진 후 정지, 측정.
5. 30스텝 완료 후 `테스트요약()` 실행, 라디오로 요약 전송, 5x5 매트릭스에
   완료 아이콘 표시.

## 스텝당 측정 로직

함수 `스텝측정(스텝번호: number)`:

1. 8x8 그리드를 **3회 반복 읍기**(노이즈 확인용). 각 셀(x,y)마다 3개
   샘플의 **중앙값(median)** 을 최종 그리드 값으로 채택.
2. 그리드 정중앙 2x2 블록(로봇 정면 기준, 예: x=3..4, y=3..4)에 대해
   3회 샘플 중 **최대-최소 범위(노이즈)** 를 계산 → `노이즈C`.
3. 중앙 2x2의 median 값들의 median을 `중심값`(라이다 대표값, mm)으로
   계산.
4. `maqueenPlusV2.readUltrasonic(P13, P14)`로 초음파 1회 읽기(cm) →
   mm로 환산 → `초음파값`.
5. `기대거리mm = 스텝번호 * 10`.
6. 오차 계산: `dLidar = 중심값 - 기대거리mm`, `dUson = 초음파값mm -
   기대거리mm` (둘 다 0/무효 측정이면 오차는 표시하지 않고 `N/A`로
   표기).
7. 점프 판정: `이전중심값`(직전 스텝의 `중심값`, 모듈 전역 변수)과 비교.
   `|중심값 - 이전중심값| > 큐브테스트_점프임계mm`(기본 25mm, 기대
   변화량 10mm의 2.5배)이면 `JUMP`로 표시하고 점프 스텝 목록에 기록.
8. 라디오 전송 (총 9줄):
   - 헤더: `STEP {s} cm={s} exp={기대거리mm}mm lidarC={중심값}mm
     uson={초음파값mm}mm dLidar={dLidar} dUson={dUson}
     noiseC={노이즈C} {JUMP 또는 빈 문자열}`
   - 그리드 8줄: `ROW{y}: {v0} {v1} ... {v7}` (median 그리드 값, mm,
     0 = 미인식)
9. `이전중심값`을 현재 `중심값`으로 갱신.
10. 다음 비교를 위해 스텝 결과(`스텝번호`, `중심값`, `dLidar`,
    `dUson`, `JUMP 여부`)를 배열에 누적 저장.

## 종료 후 요약 (`테스트요약()`)

축적된 31개 스텝 결과를 바탕으로:

- **최소 인식 거리 추정**: `중심값`이 처음으로 0이 아니고, 그 다음
  스텝에서도 0이 아니며 두 값의 차이가 `큐브테스트_점프임계mm` 이내로
  안정된 첫 스텝의 `기대거리mm`.
- **최대 오차 스텝**: `|dLidar|`가 가장 큰 스텝과 그 값.
- **점프 스텝 목록**: `JUMP`로 기록된 모든 스텝 번호.

이 세 가지를 라디오로 각각 한 줄씩 전송하고, 5x5 매트릭스에
`basic.showIcon(IconNames.Yes)`를 표시해 테스트 완료를 알린다.

## 에러 처리

- 라이다/초음파가 0 또는 비정상 값을 반환해도 스크립트는 멈추지 않고
  해당 스텝을 `dLidar`/`dUson` = `N/A`로 기록하며 계속 진행한다(센서
  한계를 찾는 것이 테스트의 목적이므로 무효값 자체가 유효한 데이터).
- `pidControlDistance` 후진이 예상보다 일찍 멈추는 경우(엔코더 슬립
  등)는 이번 설계에서는 별도 보정을 하지 않는다 — 이 테스트는 정밀
  주행 검증이 아니라 센서 응답 특성 파악이 목적이며, 실제 이동거리와
  명령 거리의 오차는 추후 별도로 다룬다.

## 테스트 / 검증 방법

- 큐브를 정확히 측정 가능한 줄자/마퀸 휠 회전수 등으로 0, 10, 20,
  30cm 지점에 표시해 두고, 스크립트 실행 중 LCD 또는 줄자로 로봇의
  실제 위치가 예상 스텝과 일치하는지 육안으로 대조한다.
- 별도 수신기 마이크로비트(기존 라디오 디버그 콘솔, 그룹 77)로 전체
  31스텝 × 9줄 로그를 캡처해, 사용자가 그대로 복사해 전달하면 분석할
  수 있다.
- 1회 실행 결과를 통해: (1) 센서 최소 인식 거리, (2) 정상 작동 구간의
  평균 오차, (3) 튀는 지점의 패턴(특정 거리 구간에서만 발생하는지)을
  확인하는 것이 성공 기준이다.
