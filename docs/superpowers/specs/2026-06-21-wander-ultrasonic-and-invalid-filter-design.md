# 라이다 무효값(4000) 필터 + 전방 초음파 통합 설계 (AUTONOMOUS_WANDER_EXAMPLE.md)

## 배경 / 목적

실제 라디오 로그 분석으로 두 가지 구조적 문제를 확인했다.

1. **라이다 무효값 미필터링**: `AUTONOMOUS_WANDER_EXAMPLE.md`는 라이다
   raw 값에서 `0`만 무효로 거르고, 센서가 "감지 안 됨/범위 밖"일 때
   보내는 `4000`은 그냥 진짜 거리값처럼 취급한다. 실제 로그에서 라이더
   최하단쪽 raw 표본(AY7)이 10번 중 8번 4000이었다. 이런 지점이
   `바닥보정()`의 기준값(median 기반) 계산에 들어가면 기준값 자체가
   4000으로 잘못 잡힐 수 있다(`PANEL_CALIBRATION_TEST.md`의 기울기보정
   모드에서 이미 같은 버그를 `라이다무효값mm=4000` 상수로 고친 전례가
   있음 — 이번엔 메인 코드에 동일한 수정을 적용한다).
2. **전방 초음파 미활용**: 메인 코드는 전방 초음파(P13 TRIG / P14 ECHO,
   `maqueenPlusV2.readUltrasonic`)를 전혀 쓰지 않고 8x8 매트릭스 라이다
   만으로 장애물을 감지한다. 라이더가 기울어져 있으면(현재처럼) 특정
   행의 신뢰도가 떨어질 수 있는데, 초음파는 그 영향을 받지 않는 별도
   센서이므로 정면 비상정지의 보조 안전장치로 쓸 수 있다.

## 비목표 (Out of scope)

- 라이다 raw 값으로부터 정확한 물리적 기울기 각도를 역산하지 않는다 —
  이 센서의 행별 정확한 화각 스펙을 검증된 자료로 확인할 수 없어서,
  대신 이미 있는 상대 비교 방식(`기준값` 베이스라인 보정)이 정확한
  각도를 몰라도 동작하도록 하는 데 집중한다.
- 초음파를 좌/우 회전 방향 결정(`구역점수`/`탐색점수`)에는 쓰지 않는다
  — 초음파는 정면 1방향만 측정하므로, 이미 정면 구역을 보는 라이다보다
  방향 결정에 추가 정보를 주지 못한다. 정면 비상정지 판단에만 쓴다.
- `lcd레이더맵표시()`(8x8 LCD 시각화를 5x5 LED로 바꾸는 작업)는 별도
  스펙으로 분리한다(아직 미작성).
- 부팅 시 라이더 기울기 보정 흐름(A/B 버튼 의미 변경)은 별도 스펙
  (`2026-06-21-wander-boot-tilt-calibration-design.md`)에서 이미
  다뤘다 — 이 스펙은 그것과 독립적으로 적용 가능하다.

## 변경 1: 라이다 무효값 필터

라이다 raw 값을 읍는 진입점은 코드 전체에 `거리읽기(index)`(라인
332-334)와 `지점읽기(x, y)`(라인 407-409) 두 곳뿐이다(`grep
matrixPointOutput` 확인). 이 두 곳에서만 한 번 정규화하면, 이미 `0`을
무효로 처리하고 있는 모든 다운스트림 함수(`트인거리값`, `거리색`,
`막힌샘플인가`, `구역최소`, `높이최소`, `바닥보정`의 표본 수집,
`각도진단`에 넘기는 리스트 구성)가 자동으로 4000도 같이 걸러내게
된다 — 개별 함수를 손댈 필요가 없다.

```typescript
const 라이다무효값mm = 4000

function 유효거리(원시값: number): number {
    return 원시값 >= 라이다무효값mm ? 0 : 원시값
}

function 거리읽기(index: number): number {
    return 유효거리(matrixLidarDistance.matrixPointOutput(라이다주소, 샘플X[index], 샘플Y[index]))
}

function 지점읽기(x: number, y: number): number {
    return 유효거리(matrixLidarDistance.matrixPointOutput(라이다주소, x, y))
}
```

`>=`(`==`가 아님)를 쓰는 이유는 펌웨어가 4000 외에 다른 큰 실패값을
보낼 가능성까지 같이 막기 위함이다.

**부수 효과**: `lcd레이더맵표시()`의 `거리색()`이 4000을 더 이상 "트인
공간(파란색)"으로 잘못 표시하지 않고 "측정 안 됨(회색, 0과 동일)"으로
표시하게 된다 — 측정 실패와 실제 트인 공간을 구분하지 못하므로 회색이
더 정확한 표현이다.

이 정규화는 기존 위험 감지 동작을 바꾸지 않는다 — 4000은 모든 안전
임계값(`긴급정지거리mm`=280 등)보다 훨씬 커서 위험으로 잘못 잡힌 적이
없었다. 유일하게 실제로 고쳐지는 건 `기준값` 베이스라인이 4000에
오염될 수 있던 부분이다.

## 변경 2: 전방 초음파를 정면 비상정지 보조 센서로 통합

`maqueenPlusV2.readUltrasonic(trig, echo)`(`maqueenPlusV3.ts:424-444`)는
cm 단위로 반환하며, 에코를 못 받으면 `0`, 3m(300cm)를 넘으면 300으로
clamp한다 — 기존 코드의 "0=무효" 관례와 그대로 맞는다.

### 새 전역 상태/상수

```typescript
const 초음파TRIG = DigitalPin.P13
const 초음파ECHO = DigitalPin.P14
let 초음파사용 = true
let 최근초음파mm = 0
```

`초음파사용`은 `LCD맵사용`과 같은 패턴 — 문제가 생기면 끌 수 있는
탈출구를 남겨둔다.

### 새 함수

```typescript
function 초음파읍기(): number {
    if (!초음파사용) return 0
    let cm = maqueenPlusV2.readUltrasonic(초음파TRIG, 초음파ECHO)
    return cm <= 0 ? 0 : cm * 10
}

function 정면최소거리(): number {
    let 라이다값 = 구역최소(1)
    let a = 라이다값 > 0 ? 라이다값 : 9999
    let b = 최근초음파mm > 0 ? 최근초음파mm : 9999
    let 결과 = Math.min(a, b)
    return 결과 == 9999 ? 0 : 결과
}
```

`정면최소거리()`는 라이다 정면 구역 최소값과 초음파 값 중 **더 가까운
(보수적인) 쪽**을 채택한다. 둘 다 무효(0)면 0을 반환해 기존 "0=무효"
관례를 유지한다.

### 기존 함수 수정

- `감시샘플읽기()`(라인 352-355) 끝에 `최근초음파mm = 초음파읍기()`를
  추가해 매 틱(FAST/CHECK 모드 공통) 초음파 값을 갱신한다.
- `정면위험()`(라인 582-594)의 마지막 줄을 `구역최소(1)` 대신
  `정면최소거리()`를 쓰도록 바꾼다:
  ```typescript
  let 정면최소 = 정면최소거리()
  return 정면최소 != 0 && 정면최소 < 긴급정지거리mm
  ```
  (그 위의 `샘플긴급인가`/`구역막힘수` 기반 체크는 그대로 둔다 — 그건
  라이다 개별 포인트 기준 점진적 판단이고, 이번 변경은 마지막의 집계
  거리 비교 한 줄만 바꾼다.)

### LCD 표시 추가

`lcd표시()`(라인 294-304)의 4번째 줄을 다음으로 바꾼다:

```typescript
lcd문자(4, 140, 54, "L" + 구역최소(0) + " F" + 구역최소(1) + " R" + 구역최소(2) + " U" + 최근초음파mm, 0xaa00aa)
```

(`" U" + 최근초음파mm` 추가, 나머지는 동일)

## 에러 처리

- 초음파가 매번 무효(0)를 반환해도(센서 미연결/고장) `정면최소거리()`는
  라이다 값만으로 정상 동작한다(기존과 동일한 동작으로 자연스럽게
  축소).
- `pulseIn` 타임아웃(최대 약 58ms)이 매 틱 걸릴 수 있는데, 이는 가까운
  장애물이 있을 때는 빠르게 끝나고(왕복 시간 비례) 정말 트인 상황에서만
  최대 지연이 발생한다 — 안전이 우선인 상황(장애물 가까움)에서는 느리지
  않으므로 별도 스로틀은 두지 않는다. 만약 실제로 루프가 느려지는 게
  체감되면 `초음파사용 = false`로 끌 수 있다(추후 필요 시 `LCD맵갱신
  간격ms`처럼 별도 주기 상수를 추가할 수 있으나, 지금은 YAGNI로 보류).

## 테스트 / 검증 방법

자동화 테스트 러너가 없으므로(MakeCode/PXT 하드웨어 펌웨어), tsc 정적
타입체크 + 실제 하드웨어 체크리스트로 검증한다.

- 부팅 후 빈 바닥에서 `기준값`이 정상적으로 잡히는지(예전과 동일한
  거동) 확인 — 4000 필터 적용 전/후로 회피 동작이 달라지지 않아야
  한다(라이더가 멀쩡한 지점에서는).
- 라이더 앞에 손바닥처럼 라이다가 놓치기 쉬운 낮은 장애물을 댄 상태와,
  초음파만 감지할 수 있는 각도/높이의 장애물을 각각 테스트해서, 둘 중
  하나만 감지해도 정지하는지 확인.
- 초음파 연결을 빼고도(또는 `초음파사용=false`) 기존처럼 라이다만으로
  정상 회피하는지 확인.
- 주행 중 LCD에 `"L.. F.. R.. U.."` 형식으로 초음파 값이 같이 표시되는지
  확인.
- `lcd레이더맵표시()`에서 4000이었던 칸이 더 이상 파란색이 아니라
  회색(0과 동일)으로 보이는지 확인.
