# 정면 고정 장착 라이다 + 8열 회피/탈출 알고리즘 — Design

## 배경 / 문제

기존 `AUTONOMOUS_WANDER_EXAMPLE.md`는 매트릭스 라이다를 바닥 쪽으로 살짝 기울여
장착하고, "열린 바닥 기준값"을 측정해 보정하는 방식이다. 이 방식은 보정 절차(`A`
버튼), 높이 변경 모드(`A+B`), 각도진단(`ANGLE OK`/`RAISE SENSOR` 등)이 모두 필요하고,
보정이 어긋나면(`ANGLE RAISE` 등) 회피 판단 자체가 흔들리는 문제가 있었다(실제
하드웨어 테스트에서 X 아이콘 정지로 이어진 사례).

라이다를 완전히 수평·정면으로 고정 장착하면 바닥을 보지 않으므로 이런 보정 절차가
구조적으로 불필요해진다. 이 설계는 그 가정 위에서, 8x8 매트릭스 전체를 열(column)
단위로 활용해 정면 장애물에 따라 경로를 바꾸는 평상시 회피 알고리즘과, 회피가
거듭 실패할 때 360도를 굵게→세밀하게 다시 훑어 빠져나가는 탈출 알고리즘을
새로 설계한다. 기존 파일은 건드리지 않고 완전히 새로운 예제 파일로 만든다.

## 범위

- 새 파일 `AUTONOMOUS_FORWARD_LIDAR_EXAMPLE.md` 작성 (기존 `AUTONOMOUS_WANDER_EXAMPLE.md`는
  수정하지 않음)
- 라이다 입력: `matrixLidarDistance.matrixPointOutput()`으로 8x8 점 단위 직접 읍기,
  8개 열 단위 최솟거리만 사용 (라이다 외 센서—초음파 등—는 사용하지 않음)
- 보정 절차 없음: 바닥 보정, 높이 변경 모드, 각도진단 전부 제거. 장착 오차는
  안전거리 여유값으로만 흡수
- 평상시 회피: 정지→8열 스캔→가장 넓게 열린 열 그룹 선택→회전→전진, 5회 연속 실패 시
  탈출 모드 전환
- 탈출: 정면 ±90°(45도) → 정면 ±90°(15도) → 우후방 105~180°(15도) → 좌후방
  -105~-180°(15도) 4단계 굵게→세밀 360도 재탐색 (기존 파일의 검증된 코스→파인
  패턴을 새 파일에 독립적으로 재구현, 호출 의존 없음)
- 모터/PID/LCD I2C 제어는 `maqueenPlusV2` 네임스페이스의 기존 블록 재사용
- LCD/5x5 LED/무선 디버그 로그는 기존 파일과 같은 스타일로 새로 구현

## 범위 외

- 기존 `AUTONOMOUS_WANDER_EXAMPLE.md`의 어떤 함수도 수정/참조하지 않음
- 초음파 센서 사용 안 함
- 라이다 높이 변경 모드, 바닥 보정, 각도진단(`ANGLE OK` 등) — 전부 미구현
- 좌/우/정면 3구역 단순화 — 8열 단위 세밀 평가로 대체

## 하드웨어 가정

- 라이다는 수평(바닥과 평행), 정면을 보도록 고정 장착. 설치 각도 보정 절차 없음 —
  약간의 미세 오차는 있을 수 있으나 별도 캘리브레이션으로 보정하지 않고 안전거리
  여유값(`정지거리mm`, `안전거리mm`)으로 흡수한다.
- 매트릭스 라이다 FOV는 60도(`matrixLidarDistance.ts` 모듈 설명 기준). 8개 열이
  FOV를 균등 분할하므로 열당 각도폭은 `60 / 8 = 7.5도`.
- 열 인덱스(0~7) → 각도 변환: `열각도(col) = (col - 3.5) * 7.5`
  - col0 = -26.25°, col3 = -3.75°, col4 = +3.75°, col7 = +26.25°
- 센서의 "감지 없음" 센티널 값(4000 이상)은 읍는 즉시 0으로 필터링한다 — 이미
  `라이다무효값mm` 필터링 패턴이 기존 파일에 있으므로 같은 컨벤션을 새 파일에도
  적용한다. 필터링된 0은 "장애물 없음(완전히 열림)"으로 취급한다.

## 데이터 구조: 8열 최솟거리

```typescript
function 열최소읍기(col: number): number {
    let 최소 = 0
    for (let row = 0; row < 8; row++) {
        let raw = matrixLidarDistance.matrixPointOutput(라이다주소, col, row)
        let mm = raw >= 라이다무효값mm ? 0 : raw
        if (mm > 0 && (최소 == 0 || mm < 최소)) {
            최소 = mm
        }
    }
    return 최소
}

function 전체열스캔(): number[] {
    let 결과: number[] = []
    for (let col = 0; col < 8; col++) {
        결과.push(열최소읍기(col))
    }
    return 결과
}
```

- 0 = 해당 열에 8개 행 전체가 "감지 없음"(완전히 열린 방향)
- 양수 = 8개 행 중 가장 가까운 장애물 거리(mm)
- 한 틱에 8열 × 8행 = 64회 I2C 점 읍기가 필요하므로, 평상시 주행에서는 정면 판단에
  필요한 즉시성과 폴링 비용의 균형을 맞추기 위해 루프 주기를 기존 파일과 비슷한
  수준(`루프대기ms`)으로 유지한다. (성능 이슈가 실제로 발견되면 이후 별도 작업으로
  열 일부만 우선 읍는 최적화를 검토할 수 있으나, 이번 설계의 범위는 아니다.)

## 평상시 회피 알고리즘

```typescript
function 정면안전(): boolean {
    let c3 = 열최소읍기(3)
    let c4 = 열최소읍기(4)
    return (c3 == 0 || c3 >= 정지거리mm) && (c4 == 0 || c4 >= 정지거리mm)
}

function 최선열찾기(거리목록: number[]): number {
    let 최선시작 = -1
    let 최선길이 = 0
    let 현재시작 = -1
    let 현재길이 = 0
    for (let col = 0; col < 8; col++) {
        let 열림 = 거리목록[col] == 0 || 거리목록[col] >= 안전거리mm
        if (열림) {
            if (현재길이 == 0) 현재시작 = col
            현재길이 += 1
            if (현재길이 > 최선길이) {
                최선길이 = 현재길이
                최선시작 = 현재시작
            }
        } else {
            현재길이 = 0
        }
    }
    if (최선길이 < 최소그룹폭열수) return -1
    let 중심열 = 최선시작 + (최선길이 - 1) / 2
    return 중심열
}

function 회피시도(): boolean {
    let 거리목록 = 전체열스캔()
    let 목표열 = 최선열찾기(거리목록)
    if (목표열 < 0) {
        실패연속 += 1
        로그("AVOID NO GAP F" + 실패연속)
        return false
    }
    let 목표각 = Math.round((목표열 - 3.5) * 7.5)
    maqueenPlusV2.pidControlAngle(목표각, maqueenPlusV2.MyInterruption.NotAllowed)
    maqueenPlusV2.pidControlDistance(적응전진거리cm, maqueenPlusV2.MyInterruption.NotAllowed)
    if (!정면안전()) {
        실패연속 += 1
        로그("AVOID STILL BLOCKED F" + 실패연속)
        return false
    }
    실패연속 = 0
    return true
}
```

- 메인 루프는 매 틱 `정면안전()`을 확인한다. 안전하면 그대로 직진. 안전하지 않으면
  정지하고 `회피시도()`를 호출한다.
- `회피시도()`가 `false`를 반환할 때마다 `실패연속`이 증가하고, **5에 도달하면**
  탈출 알고리즘(`탈출360()`)으로 전환한다. `true`를 반환하면 `실패연속 = 0`으로
  리셋되고 평상시 주행으로 복귀한다.
- `최소그룹폭열수`(기본 2)는 1열짜리 틈을 노이즈로 보고 무시하기 위한 최소
  연속 열 개수다.
- `적응전진거리cm`은 기존 파일과 동일한 가변 전진 패턴을 따른다: `회피시도()`가
  `true`를 반환(성공)할 때마다 성공 연속 카운터를 늘리고, `전진성공증가조건`회
  연속 성공하면 `전진성공증가cm`만큼 늘려 `최대전진거리cm`까지 키운다.
  `회피시도()`가 `false`를 반환(실패)하면 성공 연속 카운터를 0으로 리셋하고
  `전진실패감소cm`만큼 줄여 `최소전진거리cm`까지 낮춘다.
- 전역 상태 변수: `실패연속`(회피 연속 실패 횟수), `상태`(현재 동작 상태 문자열),
  `마지막판단`(LCD/로그용 마지막 판단 사유), `마지막탐색점수`(탈출 탐색 단계의
  최고 점수) — 기존 파일과 같은 역할의 전역 변수를 새 파일에 독립적으로 선언한다.

## 탈출(360도 굵게→세밀) 알고리즘

기존 `AUTONOMOUS_WANDER_EXAMPLE.md`에서 검증된 코스→파인 다단계 360도 탐색
패턴(누적 오차 없는 단계별 되돌리기 회전)을 이 새 파일에 독립적으로 재구현한다.
점수 함수는 14개 고정 샘플이 아니라 8열 최솟거리를 가중합산한다.

```typescript
function 탐색점수계산(): number {
    let 거리목록 = 전체열스캔()
    let 점수 = 0
    for (let col = 0; col < 8; col++) {
        let 거리 = 거리목록[col] == 0 ? 탐색점수상한mm : Math.min(거리목록[col], 탐색점수상한mm)
        점수 += 거리 * 열가중치[col]
    }
    return 점수
}

function 열각도순회탐색(각도목록: number[]): number {
    let 최고점수 = -999999
    let 최고각 = 각도목록[0]
    maqueenPlusV2.pidControlAngle(각도목록[0], maqueenPlusV2.MyInterruption.NotAllowed)
    for (let i = 0; i < 각도목록.length; i++) {
        let 후보각 = 각도목록[i]
        let 점수 = 탐색점수계산()
        if (점수 > 최고점수) {
            최고점수 = 점수
            최고각 = 후보각
        }
        로그("ESCAPE DIR " + 후보각 + " SCORE " + 점수)
        if (i < 각도목록.length - 1) {
            maqueenPlusV2.pidControlAngle(각도목록[i + 1] - 각도목록[i], maqueenPlusV2.MyInterruption.NotAllowed)
        }
    }
    let 복귀각 = 최고각 - 각도목록[각도목록.length - 1]
    마지막탐색점수 = 최고점수
    maqueenPlusV2.pidControlAngle(복귀각, maqueenPlusV2.MyInterruption.NotAllowed)
    basic.pause(300)
    return 최고각
}

function 탈출360(): boolean {
    상태 = "ESCAPE"
    maqueenPlusV2.pidControlStop()

    let 최고각 = 열각도순회탐색(탈출각도)
    if (마지막탐색점수 >= 탈출최소점수) return true
    maqueenPlusV2.pidControlAngle(-최고각, maqueenPlusV2.MyInterruption.NotAllowed)

    상태 = "ESCAPE-FINE"
    최고각 = 열각도순회탐색(탈출각도세밀)
    if (마지막탐색점수 >= 탈출최소점수) return true
    maqueenPlusV2.pidControlAngle(-최고각, maqueenPlusV2.MyInterruption.NotAllowed)

    상태 = "ESCAPE-BACK-R"
    최고각 = 열각도순회탐색(탈출각도세밀후방우)
    if (마지막탐색점수 >= 탈출최소점수) return true
    maqueenPlusV2.pidControlAngle(-최고각, maqueenPlusV2.MyInterruption.NotAllowed)

    상태 = "ESCAPE-BACK-L"
    최고각 = 열각도순회탐색(탈출각도세밀후방좌)
    if (마지막탐색점수 >= 탈출최소점수) return true

    로그("ALL 360 ESCAPE STAGES FAILED -> NO ESCAPE")
    return false
}
```

- 각 단계 성공 시 그 방향을 향한 채로 종료하고 `실패연속 = 0`, 평상시 회피 루프로
  복귀한다.
- 4단계 모두 실패하면 `마지막판단 = "NO ESCAPE"`, `basic.showIcon(IconNames.No)`로
  표시하고 정지한다(기존 파일의 `"NO SAFE DIR"` 실패 패턴과 동일한 형태).
- 각도 배열:
  - `탈출각도 = [-90, -45, 0, 45, 90]`
  - `탈출각도세밀 = [-90, -75, -60, -45, -30, -15, 0, 15, 30, 45, 60, 75, 90]`
  - `탈출각도세밀후방우 = [105, 120, 135, 150, 165, 180]`
  - `탈출각도세밀후방좌 = [-105, -120, -135, -150, -165, -180]`

## 시작 흐름 / 상태 표시

- 보정 절차가 없으므로 `A`(보정), `A+B`(높이 변경) 버튼 동작은 만들지 않는다.
  `B` 한 번 → 3초 카운트다운 → 평상시 회피 루프 시작.
- LCD: 8열 최솟거리 요약(예: 4그룹으로 압축한 막대 표시 또는 숫자), 현재 상태
  (`DRIVE`/`AVOID`/`ESCAPE`/`ESCAPE-FINE`/`ESCAPE-BACK-R`/`ESCAPE-BACK-L`),
  마지막 판단 사유를 기존 파일과 동일한 5줄 고정 좌표 갱신 방식으로 표시한다.
- 5x5 LED: 8열 데이터를 5칸으로 다운샘플링(예: col0-1→칸0, col2-3→칸1,
  col3-4→칸2(중앙), col4-5→칸3, col6-7→칸4)해 근접도를 밝기로 표시한다.
- 무선 디버그 로그(`로그()`)는 기존 파일과 동일한 분할 전송 컨벤션으로 새로
  구현한다.

## 기본 조정값

| 변수 | 기본값 | 의미 |
|---|---:|---|
| `정지거리mm` | 250 | 중앙 열(col3,4)이 이 거리 미만이면 즉시 정지 |
| `안전거리mm` | 400 | 회피 시 "열렸다"고 판단하는 열별 기준거리 |
| `최소그룹폭열수` | 2 | 이 개수 미만으로 연속 열린 열은 노이즈로 무시 |
| `적응전진거리cm` | 6 (4~10 가변) | 회피 후 전진 거리, 성공/실패에 따라 가변 (기존 파일과 동일한 가변 전진 패턴) |
| `실패연속한계` | 5 | 회피 시도 연속 실패 시 탈출 모드 전환 기준 |
| `탐색점수상한mm` | 1000 | 점수 계산 시 거리값 상한(너무 먼 거리가 점수를 과대하게 키우지 않도록 클리핑) |
| `열가중치` | `[1, 1.5, 2, 3, 3, 2, 1.5, 1]` | 탐색 점수 계산 시 중앙 열에 더 큰 가중치 |
| `탈출최소점수` | 8000 | 탈출 탐색에서 이 점수 이상이면 그 방향을 채택 |
| `라이다무효값mm` | 4000 | 이상이면 "감지 없음"으로 0 처리 |

## 검증

자동 테스트 러너가 없는 프로젝트라 검증은 기존 파일과 같은 패턴을 따른다.

1. 정적: `typescript` 코드 블록 추출 후 `tsc --noEmit` 타입 체크
2. 의사실행 트레이스: `최선열찾기()`가 다양한 거리목록 패턴(완전 개방, 중앙만 막힘,
   좌/우만 열림, 전부 막힘)에 대해 기대한 열 인덱스를 반환하는지 손으로 검증.
   `열각도순회탐색()`의 회전량 누적/복귀가 새 각도 배열에서도 오차 없이 원위치로
   돌아오는지 확인(기존 파일에서 이미 검증된 동일한 수학 패턴 재사용).
3. 하드웨어 체크리스트(사람이 실행): 라이다를 수평 정면으로 장착한 상태에서
   (a) 정면이 트인 공간에서 평상시 직진이 잘 되는지, (b) 한쪽이 좁게 막힌 상황에서
   반대쪽 열린 열로 회전·회피하는지, (c) 좁은 복도/모서리에서 5회 연속 회피
   실패 후 탈출 모드가 켜지는지, (d) 완전히 사방이 막힌 상태에서 4단계 모두
   실패해 `"NO ESCAPE"`가 뜨는지 확인.
